01
Problem Statement

Why Traditional Trackers Fail
for Irregular Cycles

Most period trackers assume a fixed 28-day cycle. For the 26% of women with PCOS or irregular periods, this leads to wildly inaccurate predictions.

❌

Fixed Cycle Length

Traditional apps use a single average. A 28-day average is meaningless for someone with cycles ranging 22–65 days.

❌

Outlier Filtering

Most apps discard "abnormal" long cycles (>45d). But for PCOS users, these ARE the pattern and must be respected.

❌

Single-Point Predictions

Showing one predicted date with high irregularity creates false precision. Users need a confidence window.

02
Our Approach

Lunara's Design Philosophy

Every design decision in our intelligence engine respects PCOS and irregular patterns as valid data — not noise to filter out.

  • ①
    No Outlier Filtering — Long cycles (>45d) are weighted down, never discarded. This preserves PCOS/anovulatory patterns in the data.
  • ②
    Recency Weighting — Recent cycles influence predictions exponentially more than older cycles via a decay factor (λ = 0.80).
  • ③
    Symptom Correlation — Pre-menstrual marker symptoms can shift the predicted date by ±3 days in real-time.
  • ④
    Adaptive Luteal Phase — Instead of assuming 14 days, the engine learns the actual luteal length from ovulation-proximal symptoms.

CycleGapType _classifyGap(int days) {
  if (days <= 35) return CycleGapType.normal;
  if (days <= 45) return CycleGapType.extended;
  if (days <= 90) return CycleGapType.extendedLutealTransition;  // PCOS common
  return CycleGapType.anovulatoryGap;  // Likely no ovulation occurred
}
05
Core Algorithm

Exponential Decay Weighted Average

The heart of Lunara's prediction engine. Each cycle gap gets two weights: recency decay and gap-type penalty, multiplied together.

📐 Formula

For each cycle gap i (most recent first):
recency_weight = 0.80 (n − 1 − i)
type_weight = { 1.0, 1.0, 0.5, 0.25 }
combined = recency × type

effective_length = Σ(gap × combined) / Σ(combined)

🎯 Example: PCOS Scenario

Cycle history: 32, 28, 72, 35, 30 days

The 72-day gap gets classified as ExtendedLutealTransition (0.5× weight) AND decayed by recency — so it barely pulls the average up, while the recent 35 and 30-day cycles dominate.

double _computeWeightedLength(
  List<int> gaps, List<CycleGapType> types
) {
  double totalWeight = 0;
  double weightedSum = 0;

  for (int i = gaps.length - 1; i >= 0; i--) {
    // Recency: most recent = 1.0, decays
    final recencyWeight =
      pow(0.80, (gaps.length - 1 - i));

    // Gap type penalty for long cycles
    final typeWeight = _gapTypeWeight(types[i]);

    final combined = recencyWeight * typeWeight;
    weightedSum += gaps[i] * combined;
    totalWeight += combined;
  }

  return weightedSum / totalWeight;
}

Variable Window

Adaptive Prediction Windows

Instead of a single date, Lunara dynamically grows the prediction window based on the user's actual cycle variability. This is critical for PCOS support.

📐 Window Margin Formula

margin = round(σ × 0.8).clamp(1, 7)

Window = [prediction − margin, prediction + margin]

🔔 isVariableWindow

When confidenceScore < 0.6, the UI automatically switches from showing a single date to showing a date range:

"Expected between Apr 20 – Apr 26"
instead of
"Expected on Apr 23"
